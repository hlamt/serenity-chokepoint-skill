# TOC Boundary Checklist

Complete this checklist before finalizing a chokepoint claim.

- [ ] 已定义产业价值链和研究系统边界。
- [ ] 已定义系统目标与有效产出，而非只看局部利用率。
- [ ] 已检查该现象是否只是短期短缺、库存波动或交付错配。
- [ ] 已检查候选环节是否具备 CCR 特征并影响系统吞吐。
- [ ] 已验证技术、认证、客户切换或监管导致的短期替代难度。
- [ ] 已列出制约释放路径，包括扩产、良率、替代、流程优化和需求调整。
- [ ] 已识别制约迁移的方向、时间和监测信号。
- [ ] 已检查价值是否由候选公司捕获，而非被客户、竞争者或新增资本稀释。
- [ ] 已将事实、来源观点、项目推断和待验证问题分开。
- [ ] 若使用“瓶颈”，已说明它是 supply-chain chokepoint、local bottleneck 还是 TOC-style constraint。
- [ ] 未把 Chokepoint 写成 TOC 意义上的同义概念。
- [ ] 已保留反证、风险、跟踪指标、假设失效条件和免责声明。
