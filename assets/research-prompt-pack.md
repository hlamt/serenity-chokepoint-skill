# Research Prompt Pack

These prompts produce research hypotheses, not investment conclusions. Replace bracketed fields and require dated, traceable public evidence.

## 1. A 股 AI 半导体供应链卡点投研

```text
请以【研究日期】为截止日，对 A 股 AI 半导体产业链进行供应链卡点投研。先定义研究边界、产业价值链与有效产出，再识别可能影响系统交付的候选供应链卡点。使用 TOC-informed CCR 检查，但不要把 Chokepoint Theory 自动等同于 TOC / 制约理论。

请区分事实、来源中的观点、项目推断与待验证问题，并输出：
1. 候选供应链卡点及其吞吐影响、替代难度和扩产周期；
2. 候选公司或公司类型，以及与卡点的暴露路径；
3. 证据链、证据等级、来源日期与验证路径；
4. 反证和风险清单，包括估值、替代、扩产、客户压价与制约迁移；
5. 可持续跟踪的价格、产能、交期、良率、认证、订单和库存指标；
6. 可明确推翻研究假设的失效条件；
7. 非投资建议免责声明。

没有公开证据支持的内容必须标为假设，不得形成确定性公司结论。
```

## 2. AI 基建供应链卡点分析

```text
请拆解【AI 基建范围，例如数据中心电力、网络、服务器、散热或建设交付】的价值链，定义系统目标与有效产出，并识别当前候选供应链卡点。判断每个候选项是结构性卡点、潜在 CCR、局部瓶颈、短期短缺还是热门叙事。

请输出：候选供应链卡点、候选公司或公司类型、价值捕获机制、证据链与待验证资料、风险清单、跟踪指标、制约释放和迁移路径、假设失效条件及免责声明。所有关键判断附日期和公开来源；缺失信息列为研究任务，不补造数据。
```

## 3. 判断某环节是真卡点还是短期短缺

```text
请评估【环节或产品】是真正的供应链卡点，还是短期短缺、local bottleneck 或市场叙事。先说明产业价值链边界、系统目标和有效产出，再检查需求持续性、吞吐影响、CCR 特征、替代难度、认证壁垒、扩产周期、库存、价格与交期。

请输出：分类结论及置信度、候选供应链卡点、受益或受损的候选公司类型、支持与反证证据链、仍需验证的路径、风险清单、跟踪指标、假设失效条件和免责声明。结论必须保留研究假设属性。
```

## 4. 用 TOC-informed 检查校准投研假设

```text
请对以下投研假设执行 TOC-informed 校准：【粘贴假设】。

依次检查系统边界、系统目标、有效产出、候选 CCR、吞吐影响、短期替代难度、制约利用与扩张、上下游从属关系、制约释放和制约迁移。指出原假设把局部繁忙、短期短缺或供应链卡点误写为 TOC-style constraint 的地方。

请输出：校准后的候选供应链卡点、候选公司或公司类型、证据链和证据缺口、验证路径、风险清单、跟踪指标、假设失效条件和免责声明。不要因术语校准而跳过用户真正需要的产业研究。
```

## 5. 检查中文文章中的术语误读

```text
请分析以下中文文章是否把 Chokepoint Theory 简化误读为 TOC 意义上的“瓶颈理论”：【粘贴文章或公开来源】。

逐段区分 supply-chain chokepoint、local bottleneck、TOC-style constraint、CCR、短期短缺、热门叙事与普通“卡脖子”概念。分别标注作者原始表达、可验证事实、你的术语校准与仍需查证之处。

在完成术语分析后，继续输出该文章涉及的候选供应链卡点、候选公司或公司类型、证据链、验证路径、风险清单、跟踪指标、假设失效条件和免责声明。不得把文章观点直接转成确定性投资结论。
```
