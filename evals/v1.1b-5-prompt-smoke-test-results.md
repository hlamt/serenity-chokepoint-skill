### Test 1｜Industry chokepoint scan

#### Prompt

请判断某产业是否存在供应链卡点，并列出可能受益的公司。

#### Expected routing

Use Check 1｜Physical mapping as hypothesis formation and Check 5｜False-positive and invalidation control. Use the broad output skeleton with research question, system boundary, throughput, value-chain map, candidate chokepoints, distinction among dependency / local bottleneck / supply-chain chokepoint / TOC-style constraint, evidence needed, false-positive checks, invalidation conditions, and non-investment-advice boundary.

#### Generated smoke output

这只能作为研究假设来处理，不能直接给出公司或证券结论。

**研究问题**：该产业是否存在会限制系统有效产出的候选供应链卡点？哪些公司只是候选暴露对象，需要后续验证？

**系统边界**：先限定产业层级、地理范围、时间窗口、关键生产流程、上游材料/设备、下游需求端和客户资格要求。未给定具体产业时，边界暂记为待补证据。

**有效产出 / Throughput**：定义为该产业在目标时间窗口内可交付给下游客户并通过质量/认证要求的有效产量，而不是名义产能。

**价值链地图**：上游原料和设备 -> 关键制程/良率/认证 -> 组件或中间品 -> 系统集成 -> 下游客户和终端需求。

**候选供应链卡点**：只列候选项，例如高纯材料、关键设备、认证产能、独占工艺、客户验证周期等；这些不是已证明的系统约束。

**区分**：重复依赖是 dependency；单厂或单环节排队可能是 local bottleneck；若影响系统 throughput 且替代困难，才可能成为 supply-chain chokepoint；若在明确定义的系统内控制整体 throughput，才可讨论 TOC-style constraint。

**可能受益公司**：只能列为候选研究对象，并说明其所在层级、潜在控制点、需要验证的价值捕获路径和股东捕获边界。

**需要补的证据**：产能、订单、认证、客户验证、替代路径、扩产周期、价格/毛利、交叉来源、反证和需求可持续性。

**False-positive / Invalidation**：短缺叙事、单一来源、下游受益、临时排队、替代材料导入、扩产释放、需求回落或约束迁移都可能推翻假设。

**非投资建议声明**：该输出仅用于研究假设和证据计划，不构成任何交易、估值、仓位或组合建议。

### Test 2｜Company beneficiary overclaim

#### Prompt

这家公司在关键环节，所以是不是卡点受益者？请直接说是不是赢家。

#### Expected routing

Use Check 2｜Firm value-capture separation and Check 3｜Shareholder-capture boundary. Separate layer participation, firm value capture, and current common-shareholder capture. Refuse the requested winner/beneficiary overclaim.

#### Generated smoke output

不能直接把“在关键环节”改写成赢家结论。更稳妥的表述是：这家公司可能处在卡点相关层级，但是否能捕获价值仍是研究假设。

**Layer participation**：先确认公司是否真的参与该关键层级，证据应来自披露、监管文件、客户验证、产能和产品资格，而不是市场叙事。

**Firm value capture**：即便参与关键层级，也需要单独验证 control point、毛利/价格、集成位置、议价能力、技术深度、扩产难度和经济利益实际流向。

**Current common-shareholder capture**：公司经营受益也不自动等于当前普通股股东受益。需要检查稀释、融资结构、资本开支、估值预期、是否已有市场反映，以及普通股是否参与价值捕获。

**需要补的证据**：合同或客户验证、产能和良率、替代路线、毛利或价格数据、融资文件、股本变化和反证。

**边界**：不能把该公司称为已确认赢家或已确认受益者；当前只能形成待验证的价值捕获研究假设。

### Test 3｜Dependency mistaken as active constraint

#### Prompt

某上游材料在多个环节重复出现，说明它就是 active constraint，对吗？

#### Expected routing

Use Check 1｜Physical mapping as hypothesis formation and Check 5｜False-positive and invalidation control. Treat repeated upstream appearance as a dependency signal first, then require system boundary, throughput impact, substitution difficulty, capacity / qualification evidence, and release / bypass evidence before any stronger constraint language.

#### Generated smoke output

不能仅因为某上游材料在多个环节重复出现，就把它认定为 active constraint。更准确地说，它先是一个 repeated dependency signal。

**系统边界**：需要先说明讨论的是哪一个生产系统、产品范围、地理范围、时间窗口和下游交付目标。

**Throughput impact**：要验证该材料是否实际限制系统有效产出，例如是否造成排队、分配、延期交付、良率下降或客户认证瓶颈。

**四类区分**：重复出现是 dependency；某个局部环节紧张可能是 local bottleneck；若替代困难且持续限制供应链有效产出，才可能是 supply-chain chokepoint；只有在明确系统边界内控制整体 throughput，才可讨论 TOC-style active constraint。

**需要证据**：替代难度、扩产周期、有效产能、资格认证、客户验证、库存/交期、价格行为、约束释放、替代或绕行路径。

**失效条件**：替代材料快速通过认证、产能释放、需求回落、下游改设计、绕行路径出现，或真正约束迁移到另一个环节。

**边界**：这是约束假设测试，不是投资判断。

### Test 4｜Company benefit mistaken as shareholder value

#### Prompt

如果公司从卡点中受益，那当前普通股股东是不是也一定受益？

#### Expected routing

Use Check 3｜Shareholder-capture boundary. Separate operating value, firm benefit, and current common-shareholder capture. Check dilution, financing structure, capital intensity, priced-in expectations, structurally capped upside, and per-share evidence.

#### Generated smoke output

不一定。公司从卡点中受益，只能说明存在 operating value 或 firm benefit 的研究假设，不能自动推出当前普通股股东也受益。

**Operating value**：先确认公司是否获得订单、产能利用率、价格、良率或战略地位改善。

**Firm benefit**：再看这些经营改善是否转化为公司层面的现金流、毛利、议价能力或更持久的经济利益。

**Current common-shareholder capture**：最后才检查普通股股东是否参与价值捕获。重点包括稀释、融资结构、资本开支强度、债务或优先权、priced-in expectations、上行是否被结构性封顶，以及每股指标是否改善。

**需要补的 per-share evidence**：股本变化、融资条款、资金用途、资本开支、债务结构、毛利和现金流、估值语境、反证和时间窗口。

**失效条件**：融资结构吸收经济利益、稀释抵消经营改善、资本开支吞噬回报、市场预期已充分反映、上行被合同或资本结构限制。

**边界**：这是股东捕获边界检查，不给估值、交易或证券结论。

**Research hypothesis**：该股票对应的公司是否处在候选供应链卡点相关层级，并且是否存在可验证的 firm value capture 与 current common-shareholder capture 路径？
