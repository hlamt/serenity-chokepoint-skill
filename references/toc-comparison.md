# TOC Comparison

TOC / 制约理论在本项目中作为投研判断的系统性校准框架。它帮助研究者检查一个候选供应链卡点是否真的影响系统有效产出，以及制约在被缓解后可能迁移到哪里；它不替代产业证据，也不与 Chokepoint Theory 同义。

## Calibration Dimensions

| TOC concept | Use in this project | Investment-research implication |
| --- | --- | --- |
| 系统目标 | 明确研究对象追求的结果，例如可交付算力、合格晶圆或可运营产能。 | 防止把局部繁忙误判为全链卡点。 |
| 有效产出 | 定义系统真正增加的可交付价值，而非单一设备利用率或产量。 | 检查稀缺环节是否影响终端交付与收入实现。 |
| CCR | 识别控制或可能控制流动的产能约束资源。 | 观察排队、交期、稼动率、良率、认证与投料顺序。 |
| 短期难以替代 | 检查技术、认证、客户切换、供应集中与资产专用性。 | 判断短缺是否可能持续并形成价值捕获。 |
| 制约释放 | 记录扩产、提价抑制需求、替代、良率提升与流程优化。 | 估计卡点持续期与供需修复路径。 |
| 制约迁移 | 预判当前环节缓解后，下一个限制有效产出的环节。 | 避免把阶段性超额收益线性外推。 |
| 控制投料 | 让上游投料与制约节拍匹配，避免在制约前堆积无效在制品。 | 检查订单、库存和产能利用率是否被重复计算或虚高。 |

## TOC-Informed CCR Check

1. Define the value-chain boundary and system goal.
2. Define measurable throughput.
3. Ask whether the candidate node controls throughput rather than merely running at high utilization.
4. Inspect queue, lead time, yield, qualification, substitution, and scheduling evidence.
5. Identify exploitation, subordination, and elevation actions already underway.
6. Estimate when the constraint may be released and where it may migrate.

TOC language raises the standard of proof. It does not turn a candidate chokepoint into a confirmed investment opportunity.
