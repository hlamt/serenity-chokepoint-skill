# Targeted X Original Expression Gap Search v1.0G

## 1. Purpose

This file translates the bounded source gaps identified in `docs/method-claim-review-v1.0f.md` into targeted X search locators and candidate source-lead review tasks for Serenity / `@aleabitoreddit`.

The search design is limited to specific wording, counterevidence, and scope gaps associated with MCC-0001 through MCC-0005. MCC-0006 remains outside the method layer and does not require an X original-expression search.

- This is not corpus.
- This is not source records.
- This is not method claims.
- This is not validated claims.
- This is not TOC calibration.
- This is not investment advice.

An X search locator is not a source record. An X search result is not corpus. A post may become an input to a later Source Lead Review Packet only after a human reviewer confirms the post URL, author, visible date, text, context, and provenance. No post can directly become a validated method claim.

Public-access review on 2026-06-23 found that the focused X search page redirected to a login flow. A small set of targeted public-web index queries did not return a stable, reviewable `x.com/aleabitoreddit/status/...` URL. No login was attempted, no visibility control was bypassed, and no post text was inferred from search terms.

## 5. Prioritized manual review list

1. **XLOC-002 — `bottleneck`**: highest chance of finding direct recurrence, rejection, or narrowing of the two-layer language behind MCC-0001.
2. **XLOC-003 — `constraint`**: most useful for distinguishing an active system constraint from a dependency map or local bottleneck.
3. **XLOC-010 — `InP`**: tightly linked to the existing reviewed expression and may reveal whether backward physical tracing recurs.
4. **XLOC-009 — `indium`**: useful for checking whether Serenity treats a raw material as a true constraint, a candidate dependency, or a qualified premise.
5. **XLOC-012 — `photonics`**: may identify cross-case recurrence or counterexamples within the same thematic domain.
6. **XLOC-014 — `"value capture"`**: most direct test for explicit separation of operating scarcity and firm economics.
7. **XLOC-015 — `dilution`**: tests whether common-shareholder capture is treated separately from company capacity or demand.
8. **XLOC-019 and XLOC-021 — `capacity` / `qualification`**: useful for narrowing the announcement-versus-effective-capacity governance rule.
9. **XLOC-025 — `evidence`**: best direct locator for source sufficiency, hierarchy, and corroboration language.
10. **XLOC-026 — `invalidation`**: prioritize posts that state disconfirming conditions; exclude trading-only or weakly related uses.

For every manually reviewed result, record negative findings as well as positive ones. A result should be excluded when the author is not confirmable, the date or permalink is unstable, the visible text is incomplete, the term is only incidental, the post is a repost without original commentary, or the content cannot be reviewed without bypassing access controls.
